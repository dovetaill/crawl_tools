# 优化脚本计划（去除 WARP 与 Crawl4AI）

## 1. 计划元信息

- 适用仓库：`/home/wwwroot/crawl_tools`
- 方案确认结果：`1A 2B 3A 4B 5B`

## 2. 目标与边界
### 2.1 目标
- 从项目中彻底移除 `WARP` 与 `Crawl4AI` 相关实现与文档描述。
- 重写 `README.md` 为“参考手册型（2B）”文档，覆盖安装、运行、排障、参数细节。
- 生成单一 `docker-compose.yml`（3A），仅围绕 `edge + flaresolverr`。
- 明确 `install_docker.sh` 使用方式，并让 `install.sh` 在未安装 Docker 时可交互安装；已安装时可交互选择是否更新（默认不更新，4B）。
- 提供“原生终端方向键菜单”（5B），无需额外第三方交互工具。

### 2.2 非目标
- 不引入新的第三方交互依赖（如 `whiptail`、`dialog`、`fzf`、`gum`）。
- 不新增与当前需求无关的新服务或多 compose 分层方案。
- 不在本阶段做业务功能扩展（仅围绕脚本/部署/文档优化）。

## 3. 当前状态调研结论（Step 1 结果）
- 当前脚本为参数驱动，暂无方向键交互菜单实现。
- `install.sh` 内嵌生成 `docker-compose.yml` 与 `docker-compose.warp.yml`，且存在 `warp on/off`、`SERVICE_MODE`、`crawl4ai` 分支。
- `install_docker.sh` 当前为直接安装流程，未提供“已安装时是否更新”的策略控制。
- `README.md` 仍以 `WARP + Crawl4AI` 为主线，需整体重构。

## 4. 最终设计决策（已确认）
### 4.1 设计要点 1（1A）
- 采用“彻底移除”策略：
- 删除 `WARP`、`Crawl4AI` 相关 compose、env、命令分支、文档与示例。
- 产物聚焦：`edge + flaresolverr`。

### 4.2 设计要点 2（2B）
- `README.md` 采用“参考手册型”结构，包含：
- 架构说明、安装方式、交互菜单使用、配置说明、运维命令、验证方法、故障排查、升级与卸载。

### 4.3 设计要点 3（3A）
- 采用“单 compose 文件”：
- 仅保留 `docker-compose.yml`。
- 不使用 `profiles`、`override`、`warp` 附加编排文件。

### 4.4 设计要点 4（4B）
- 采用“交互 + 参数并存”：
- 交互模式（默认）：检测 Docker 状态并提问。
- 未安装：询问是否安装 Docker。
- 已安装：询问是否执行替换更新（默认 No）。
- 非交互参数（可选）：支持 `--yes` / `--update-docker` 等覆盖默认行为。

### 4.5 设计要点 5（5B）
- 采用“原生方向键菜单”：
- 使用 `read -rsn` 捕获按键，`tput` 控制光标与刷新。
- 支持 `↑/↓` 选择、`Enter` 确认、`q`/`Ctrl+C` 退出。
- 非 TTY 自动回退到数字输入模式。

## 5. 目标架构（优化后）
```text
Internet <--(BasicAuth)--> edge(Nginx) <--internal network--> flaresolverr
```

- `edge`：公网入口，BasicAuth，反代 `flaresolverr`。
- `flaresolverr`：核心服务，内网暴露给 `edge`。
- 网络：统一单一 bridge 网络（沿用 `aio-proxy-net` 或在计划执行时统一命名）。

## 6. 关键流程设计
### 6.1 install.sh 交互流程（高层）
1. 环境预检：root/sudo、OS、基础命令可用性。
2. Docker 检测：
- 若未安装：提示“是否安装 Docker”，默认选“是”。
- 若已安装：提示“检测到已安装，是否替换更新 Docker”，默认选“否”。
3. 采集部署参数（用户名/密码/端口等）。
4. 渲染并写入：
- `.env`
- `docker-compose.yml`
- `nginx.conf`
- `manage.sh`
5. 启动服务并输出访问地址与运维命令。
6. 给出验证建议（`curl` 样例与健康状态检查）。

### 6.2 方向键菜单设计（高层）
1. 初始化菜单状态：当前索引、选项数组、默认选项。
2. 渲染菜单：高亮当前项（反色或前缀符）。
3. 监听输入：
- `ESC [ A` => 上移
- `ESC [ B` => 下移
- `Enter` => 返回选择
- `q`/`Ctrl+C` => 安全退出
4. 退出时恢复光标与终端状态（`trap` 清理）。
5. 非交互终端分支：自动切换为数字编号输入。

## 10. 风险与缓解
- 风险 1：方向键序列在不同终端兼容性差异。
- 缓解：加入非 TTY/不支持序列时的数字输入回退。

- 风险 2：Docker 更新行为可能影响运行环境。
- 缓解：默认“不更新”，仅在用户明确同意或传参时更新。

- 风险 3：README 与脚本偏离。
- 缓解：以脚本参数定义为单一真源，文档按执行结果回归校对。
